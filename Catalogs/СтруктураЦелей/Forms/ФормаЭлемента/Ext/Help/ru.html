Цель

Справочник предназначен для регистрации целей и целевых показателей в системе.
Целевые показатели образуют иерархическую структуру, которая отражает логическую или математическую зависимость одних целевых показателей от других. На верхнем уровне структуры находятся укрупненные цели, которые предприятие планирует достичь в будущем. Каждая из укрупненных целей состоит из множества подцелей, успешное выполнение которых и обеспечивает достижение основной цели.

Система поставляется с заполненной структурой целевых показателей и списком вариантов для их анализа.
Доступна возможность по уточнению параметров анализа, добавлению новых целевых показателей, а также созданию собственной структуры целевых показателей.
Настройку структуры целевых показателей и вариантов анализа производит администратор системы в соответствии с подготовленной руководством предприятия стратегией.

Настроенная структура целевых показателей используется для предоставления информации о деятельности предприятия в мониторе целевых показателей.

Определение наименования цели


В наименовании цели следует сформулировать бизнес-цель, которую планируется достичь. При формулировке цели следует использовать информацию о направлении ее изменений. Например: Рост прибыли, Снижение издержек и т.д. При формулировке цели рекомендуется использовать следующую форму: [Направление изменений] [Объект изменений].

Заполнение вышестоящих целей


Вышестоящая цель - это цель, для успешного достижения которой необходимо выполнение текущей цели. Информация о вышестоящей цели заполняется автоматически в том случае, если ввод новой цели производится в одном из уровней структуры целей.

Пример. Вышестоящей целью для целей Рост валовой прибыли и Снижение косвенных затрат является цель Рост прибыли. В то же время цель Рост валовой прибыли является вышестоящей для целей, находящихся на более низком уровне иерархии (Рост выручки, Снижение себестоимости товаров и т.д.).

Измеримые и неизмеримые цели

  • Измеримыми являются цели, по которым можно однозначно получить информацию о достижении цели, то есть определить целевой показатель и его целевое значение.
  • Неизмеримыми являются цели, для достижения которых требуется анализ сразу нескольких целевых показателей или определение целевого показателя невозможно.


В справочнике Структура целей неизмеримые цели используются для группировки измеримых целей.

Пример. Цель Удержание ликвидности предприятия на приемлемом уровне не может быть однозначно измерена, но ее можно разбить на три измеримые цели, для каждой из которых можно определить конкретный целевой показатель.
Способность рассчитываться по текущим долгам (текущая ликвидность)
Способность погашения задолженности в случае сложностей с реализацией запасов (срочная ликвидность)
Способность частичного срочного погашения задолженности (абсолютная ликвидность).

Заполнение дальнейшей информации о целевом показателе имеет смысл только для измеримых целей (установлен флаг Цель измеримая).

Заполнение целевого показателя


Для целевого показателя необходимо определить следующие параметры:

  • Описание показателя - определяется его полное и краткое наименование, а также вводится подробное описание. Краткое наименование показателя в дальнейшем используется для отображения информации о показателе в мониторе целевых показателей. В качестве краткого наименования следует использовать аббревиатуру, которая обычно используется для показателя в конкретном предприятии.
  • Формула расчета - перечисление данных, которые будут использоваться для расчета целевого показателя и математическая связь между ними. В поле указывается справочная информация о формуле расчета (в математическом представлении), которая используется в шаблоне расчета.
  • Целевой тренд - направление изменений показателя (направление к цели). Выбирается один из возможных вариантов: Максимизация значения, Минимизация значения или Удержание в допустимом диапазоне.
  • Шаблон расчета показателя - определение структуры данных и формулы для расчета целевого показателя. Можно выбрать шаблон расчета показателя из списка поставляемых шаблонов. Пользователь может внести свои изменения в шаблон расчета показателя. Для выбора и редактирования шаблона расчета показателя используется кнопка Редактировать.

Заполнение текстового описания


В текстовом описании целевого показателя рекомендуется указывать следующую информацию:

  • Полное наименование показателя.
  • Международное наименование показателя, если оно есть.
  • Суть его измерения, а также те аспекты бизнеса, которые он затрагивает.
  • Ограничения расчета, если они есть.
  • Способы интерпретации тех или иных значений показателя, или его поведения.

Определение размерности показателя


Размерность определяет единицы, в которых измеряется показатель. Информация заполняется в виде текста. Для суммовых показателей размерность не указывается (поле становится невидимым). Валюта, в которой будут отображаться суммовые показатели, определяется при создании шаблона расчета целевого показателя (Валюта расчета).

Настройка шаблона расчета показателя


Для выбора поставляемого шаблона расчета необходимо:

  • в карточке цели в поле нажать на кнопку Редактировать,
  • в открывшемся окне выбрать Схему компоновки данных.


При настройке шаблона расчета показателя используются схема компоновки данных. Пользователю предоставляется упрощенная схема настройки, с помощью которой пользователь может редактировать следующие настройки шаблона расчета показателя:

  • Задавать предопределенные параметры отбора данных в соответствии с параметрами, которые определены шаблоном расчета показателя и задавать те параметры отбора, некоторые из которых могут быть доступны для быстрого отбора пользователю при использовании данного шаблона (Отбор).
  • Задавать значение параметров, если это предусмотрено шаблоном расчета показателя (Параметры).


При добавлении новой цели загружается произвольная схема компоновки данных с определенным набором параметров. В соответствии с назначением цели пользователь может выбрать нужную схему компоновки данных из списка поставляемых шаблонов (Выбрать шаблон) или создать свою схему компоновки данных, используя систему компоновки данных (Редактировать схему компоновки данных).

Выбор шаблона для расчета показателя


Шаблон расчета можно изменить только в форме редактирования схемы компоновки данных (кнопка Редактировать).
Будет загружен шаблон схемы компоновки данных. В соответствии с выбранным шаблоном будет заполнена информация в списке доступных полей для отбора.

Предусмотрены следующие шаблоны расчета показателей:

Отбор данных


Список полей, по которым пользователь может производить отбор данных, определяется в настройках шаблонов на странице Отбор. При загрузке шаблона расчета список доступных полей загружается автоматически.

Пользователь, формирующий шаблоны расчета, может расширить список этих полей, добавив в список полей отбора поля из списка Доступные поля.

Для добавления дополнительных полей в список выполните следующие действия:

  • Установите курсор на нужное дополнительное поле в списке.
  • Нажмите на кнопку Выбрать вверху списка.
  • Дополнительное поле будет добавлено в список полей отбора.


Для того, чтобы это поле было доступно для осуществления отбора данных пользователем выполните следующие действия:

  • Установите курсор на поле отбора в списке.
  • Нажмите на кнопку Еще и выберите пункт Свойства элемента пользовательских настроек.
  • В появившейся форме установите флаг Включать в пользовательские настройки.
  • В поле Представление можно изменить название отбора, которое будет отображаться у пользователя. Если это поле не заполнено, то у пользователя будет отображаться то наименование, которое указано в названии отбора, представленном в списке.

Представление рекомендуется изменять в том случае, если используется вложенный реквизит (например, Номенклатура.Артикул). Представление в этом случае следует задавать без использования точки, например, Артикул.

Ограничение области применения шаблона


При настройке шаблона расчета показателя может возникнуть необходимость создать раздельные шаблоны расчета, например, по разным бизнес-регионам. Такая возможность может быть востребована в том случае, если по различным бизнес-регионам применяются различные варианты достижения цели, и ответственность за достижение цели относится к зонам ответственности различных менеджеров.

Пример. Необходимо сформировать два различных целевых показателя для бизнес-регионов Восточный и Западный. При использовании данных шаблонов каждый менеджер должен видеть информацию только по своему бизнес-региону и не иметь возможность ее изменять.

В этом случае в параметрах отбора необходимо установить отбор по конкретному региону.

Редактирование шаблона расчета показателя (схемы компоновки данных)


Шаблон расчета показателя (схема компоновки данных) может быть отредактирован только в режиме толстого клиента.

  • Для редактирования шаблона нажмите на кнопку Редактировать в форме цели.
  • В диалоговом окне упрощенных настроек нажмите на кнопку Редактировать схему компоновки данных.


Пользователь может отредактировать текст и параметры запросов с учетом требований к схемам компоновки данных для расчета целевых показателей.

Сохранение шаблона
  • После настройки схемы компоновки данных (с использованием шаблонов или используя произвольные запросы) в диалоговой форме настройки нажмите на кнопку Завершить редактирование.
  • В карточке цели данных нажмите на кнопку Записать и закрыть.


Будет сохранена информация о шаблоне расчета показателя с введенными правилами его формирования. В дальнейшем эта информация используется при настройке вариантов анализа целевых показателей.
