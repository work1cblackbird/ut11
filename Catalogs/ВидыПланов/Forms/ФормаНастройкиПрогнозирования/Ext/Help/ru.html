В форме задаются настройки прогнозирования, которые передаются во внешний сервис.

Указание периода прогнозирования

Необходимо задать количество периодов для прогнозирования и периодичность (день, неделя, месяц). Затем выбрать дату, начиная с которой требуется построить прогноз. При выборе этой даты важно, что бы имелась история продаж, примыкающая к этой дате. Например, имея продажи по январь 2022 года, невозможно построить прогноз начиная с декабря 2022 года.

Дополнительные настройки

Имеется возможность изменить настройки по умолчанию для расчета прогноза.

  • Метрика оценки качества прогноза - формула для оценки меры качества прогнозирования.
  • Количество периодов для оценки точности - сколько уже известных периодов продаж будут использованы для проверки качества прогнозирования.
  • Взвешивание объектов при подсчете общей метрики -
  • Учет потерянных продаж -
  • Сглаживание выбросов в исторических данных - чрезмерное отклонение количества продаж за определенную дату будет сглажено.
  • Рассчитывать среднее и стандартное отклонение -
  • Ускорить обучение с потерей качества - может использоваться для быстрой проверки работоспособности компонентов сервиса прогнозирования.
