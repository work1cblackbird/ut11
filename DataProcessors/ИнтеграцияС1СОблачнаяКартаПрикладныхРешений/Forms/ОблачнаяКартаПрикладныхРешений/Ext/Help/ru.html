Сервис "1С:Облачная карта прикладных решений"

Сервис "1С:Облачная карта прикладных решений" позволяет пользователям знакомиться с актуальным описанием функциональных возможностей типовых решений фирмы "1С", входящих в состав комплексной поставки "1С:Корпорация", отраслевых и специализированных решений, выпускаемых по схеме "1С-Совместно", а также получать оценку индустриального обновления корпоративных решений на базе 1С:ERP и 1С-Совместных решений.

Функциональная модель разработана в прикладном решении "1С:Система проектирования прикладных решений" в соответствии с методологией IDEF0 (Integrated Definition Function Modelling), в простой и наглядной форме описывает логическую модель решений с учетом процессов, при этом увязываются исполнители, рабочие места и информационные потоки.

Процессы

Процессы описывают реальные процессы предприятия, автоматизация которых реализована в рамках отраслевых и специализированных решений 1С-Совместно. Процессы представлены в виде отчета и отображаются с отбором по текущему объекту навигации (функция, объект данных, профиль пользователя).

Функции

Функция - совокупность действий (операций), выполняемых пользователями и внутренними механизмами конфигурации.
Список функций конфигурации определяется в результате декомпозиции - разбиения функции на более простые составляющие (функции более низкого уровня).
Функции самого низкого уровня называются Конечными и обычно являются рабочими местами конфигурации, в которых работают пользователи.

  • Элементы схемы, связанные с функцией слева, описывают входные данные, которые обычно изменяются в функции;
  • Элементы схемы, связанные с функцией справа, описывают выходные данные;
  • Элементы схемы, связанные с функцией сверху, описывают управляющие данные, которые используются для работы функции;
  • Элементы схемы, связанные с функцией снизу, описывают исполнителей функции.

Объекты данных

Объект данных - это неделимая логическая единица информации, поступающая на вход функции, в том числе в качестве управления, или формируемая функцией.

Объекты данных делятся на:

  • Входящие данные - данные, которые попадают в информационную систему извне: загружаются автоматически или вводятся оператором;
  • Исходящие данные - данные, которые система формирует для использования вне самой системы: пользователями, руководством, другими организациями и т. п. в виде отдельных файлов, отчетов, печатных документов и т. п.
  • Внутренние данные - данные, необходимые для функционирования самой информационной системы.

Объекты данных хранятся в информационной системе и могут быть извлечены из нее.
Объекту данных в информационной системе может соответствовать:

  • документ;
  • состояние (статус) документа;
  • группа однотипных документов;
  • справочник;
  • группа реквизитов справочника, содержащих специализированную информацию, потребляемую и порождаемую одной или несколькими близкими по назначению функциями;
  • отчет;
  • движения регистра и запрос к регистру системы.

Профили пользователей

Профиль пользователя - неделимая единица функциональных обязанностей пользователя, определяющая его права в информационной системе.

ВАЖНО!!! Сервис работает в интерфейсе пользователя, но для получения актуальной функциональной модели требуется наличие интернет соединения.

Индустриальное обновление корпоративных решений

Индустриальное обновление Корпоративных решений – это сервис по обновлению сложных нетиповых решений на базе "1C:ERP", состоящих из типового "1С:ERP" и интергированных модулей, реализующих отраслевые или функциональные особенности ведения деятельности предприятия, и доработок, отражающих особенности работы конкретного предприятия, выполняемых компанией "1С-ИжТиСи" с использованием инструмента "1C:Автоматизированное обновление измененных конфигураций".

Корпоративное решение представляет собой комплекс конфигураций (в том числе кастомизированных), содержащих необходимый в рамках заданных областей автоматизации функционал, и взаимодействующих между собой в соответствии с предусмотренными для соответствующего продукта схемами интеграции. В рамках Корпоративного решения допускается использование измененных в соответствии с бизнес-требованиями конфигураций, а также применение таких схем интеграции как обмен данными, объединение конфигураций.

Ключевые преимущества сервиса: сокращение сроков, повышение качества проведения обновлений для сложных нетиповых конфигураций.

Порядок работы с заявками на оценку стоимости корпоративного решения
  • Создать (или открыть сохраненную ранее) заявку 1С:ИжТиСи на корпоративное решения;
  • Выбрать продукты, которые должны быть включены в Корпоративное решение;
  • Заполнить реквизиты отправителя и отправить заявку на оценку стоимости в "1С-ИжТиСи".

По указанному в поле "E-mail отправителя" адресу поступит письмо "Заявка на оценку стоимости обновления Корпоративного решения" с перечнем выбранных решений во вложении.

В течение 1-ого рабочего дня, после поступления заявки от "1С-ИжТиСи" будет отправлено либо коммерческое предложение, либо уточненные сроки предоставления оценки.
