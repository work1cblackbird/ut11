Назначение формы.

Форма предназначена для изменения различных настроек связанных с сервисом 1С:EDI.
Можно настроить:
По каким правилам наша организация принимает заказы от клиентов.
Каким образом загружаются и обновляются документы в информационной базе при появлении или обновлении документов в сервисе.

Как начать использовать полные возможности сервиса 1С:EDI?
  • Для начала необходимо выбрать организацию, для которой будут изменяться настройки.
  • Включить использование процессов EDI.
  • Указать префикс, для документов, которые будут создаваться в сервисе для данной организации.

Какие правила приема заказов от клиентов я могу настроить?
  • Принимаю заказы - если флаг не установлен, то сервис 1С:EDI не будет принимать заказы от клиентов для выбранной организации. Если флаг установлен, то ниже можно будет настроить черный или белый список покупателей.
  • Черный список - установите значение От всех и укажите список контрагентов, от которых не хотите принимать заказы.
  • Белый список - установите значение Только от и укажите список контрагентов, от которых планируете принимать заказы.
  • Разрешить покупателям оформлять заказы в терминах своей номенклатуры - если установить данный флаг, то сервис позволит клиентам отправлять заказы вашей организации не только в терминах вашей номенклатуры, но и в терминах своей номенклатуры. При этом есть возможность ограничить список таких покупателей. Для этого установите ниже флаг Только от и укажите контрагентов, которым разрешено присылать заказы в терминах своей номенклатуры.

Какие правила я могу задать для загрузки заказов клиента из сервиса?
  • Автоматически создавать документы по новым заказам в 1C:EDI -если флаг установлен, то новые заказы клиентов будут автоматически загружаться в информационную базу.
  • Автоматически обновлять документы заказ клиента при изменении в 1C:EDI - если флаг установлен, то все изменения сделанные в сервисе по документам заказ клиента будут автоматически загружаться в информационную базу.
  • Отменять строки заказа клиента при отмене покупателем - в случае, если покупатель передаст информацию об отмене заказа в сервис 1C:EDI, то в информационной базе у такого заказа будут отменены все строки автоматически.
  • Причина отмены заказа - применимо, если установлена настройка автоматической отмены строк. Данная причина будет установлена для всех автоматически отмененых строк заказа.
  • Способ определения склада - возможность настроить, какой склад будет автоматически подставляться в заказы клиентов, загружаемые из сервиса 1C:EDI.

Какие настройки доступны для заказов поставщикам?
  • Отправлять заказы поставщикам в 1С:EDI - установите флаг, если вы планируете отправлять заказы поставщика через сервис 1С:EDI.
  • Автоматически обновлять Заказы поставщикам при изменении в сервисе 1С:EDI - если флаг установлен, то все изменения сделанные в сервисе по документам заказ поставщику будут автоматически загружаться в информационную базу.
  • Отменять строки заказа поставщику при отмене поставщиком - в случае, если поставщик передаст информацию об отмене заказа в сервис 1C:EDI, то в информационной базе у такого заказа будут отменены все строки автоматически.
  • Причина отмены заказа - применимо, если установлена настройка автоматической отмены строк. Данная причина будет установлена для всех автоматически отмененных строк заказа.

Какие операции еще можно выполнить в данной форме?
  • Зарегистрировать другую организацию в сервисе 1С:Бизнес-сеть - команда доступна при нажатии на гиперссылку Еще возможности - будет выполнен переход в форму регистрации организаций в сервисе.
  • Обновить настройки организации из сервиса 1С:EDI - возможность актуализировать настройки по организации из сервиса 1С:EDI. Данная операция также выполняется периодически в автоматическом режиме.
