Настройки торгового представителя

Определяет настройки пользователей мобильного приложения (торговых представителей): отбор данных, участвующих в обмене, а также некоторые параметры управления обменом.

Как указать торгового представителя?

Информация о торговых представителях, которые используют при работе мобильные устройства, выбирается из справочника "Пользователи". То есть торговый представитель, использующей в своей работе мобильные устройства обязательно должен быть зарегистрирован в справочнике "Пользователи".

Как быстро ввести данные о мобильном устройстве, для которого настраивается обмен?

Для быстрого ввода информации о мобильном устройстве можно указать его серийный номер. Предусмотрен также выбор мобильного устройства из списка ранее зарегистрированных мобильных устройств.

Как указать версию мобильного приложения, которая используется при обмене?

Актуальная версия мобильного приложения выбирается из справочника "Мобильные приложения и их версии".

Можно ли при обмене данными с мобильными компьютерами ограничить список передаваемой информации?

При настройке правил обмена для каждого мобильного приложения можно задать конкретные параметры отбора данных: по организации, по номенклатурным позициям, по клиентам (партнерам) торгового предприятия, а также по складам, видам цен и причинам отмены заказов. Установленные параметры обмена ограничивают как список передаваемой справочной информации, так и список документов.

При обмене данными можно регулировать процесс выгрузки контактной информации по партнерам. Процесс выгрузки контактной информации регулируется установкой соответствующего значения в поле "Использование контактной информации".

  • Не используется - контактная информация по партнерам не выгружается.
  • Используется для чтения - выгружается контактная информация, зарегистрированная в карточке партнера и информация о контактных лицах партнера. Эта информация может быть использована торговым представителем в качестве дополнительных сведений о партнере. Торговый представитель не может изменять или регистрировать новую контактную информацию.
  • Используется для чтения и записи - выгружается контактная информация, зарегистрированная в карточке партнера и информация о контактных лицах партнера. Эта информация может быть использована торговым представителем в качестве дополнительных сведений о партнере. Торговый представитель может уточнять данные по контактной информации партнера и регистрировать новую контактную информацию по партнерам. Эта информация при обмене данными будет загружена в центральную информационную базу.

Можно ли сделать так, чтобы при загрузке данных по оформленным заданиям автоматически создавались заказы для клиентов?

Для того чтобы при обмене данными автоматически создавались заказы клиентам, в форме настройки необходимо установить флаг "Автоматически создавать заказы по заданию". Заказы клиента будут автоматически созданы по тем заданиям, у которых установлен статус "Отработано". В заказе заполнятся те позиции номенклатуры, для которых заполнено поле "Количество" в задании торговому представителю.

Важно. Данная возможность может быть применена только в том случае, если используется режим работы с заданиями торгового представителя ( в настройках параметров учета в поле "Задания торговым представителям" установлено "Используются для управления торговыми представителями"). Если в поле "Задания торговым представителям" установлено "Используются торговыми представителями для планирования", то торговый представитель создает заказы клиентов самостоятельно и при обмене данными они загружаются в центральную информационную базу.
