Интеграция с 1С:Документооборотом

Назначение подсистемы

Интеграция с 1С:Документооборотом позволяет организовать единую информационную систему, позволяющую расширить возможности типовой конфигурации по работе с документами и файлами. В результате интеграции пользователям становится доступны не только структурированные данные в виде основных реквизитов, строк табличной части и т.п., но и неструктурированные - факсы, сообщения электронной почты, изображения, презентации и т.п.
Интеграция с 1С:Документооборотом предоставляет пользователям следующие возможности:

  • автоматическое создание в 1С:Документообороте различных типов документов на основании настраиваемых взаимосвязей с объектами информационной базы,
  • автоматический запуск и контроль выполнения бизнес-процессов в 1С:Документообороте непосредственно из учетной системы,
  • оперативный доступ из учетной системы к хранящимся в 1С:Документообороте файлам и документам, являющихся неотъемлемой частью объектов информационной базы.

Настройка подсистемы

Для настройки интеграции 1С:Документооборота с учетной системой в разделе НСИ и администрирование - Настройка интеграции- Документооборот требуется выполнить следующие шаги:

  • Использование интеграции с 1С:Документооборот – установить флаг Интеграция с 1С:Документооборотом.
  • Настройка адресации для настройки соединения с 1С:Документооборотом (выполняется администратором)
  • Использование согласований через 1С:Документооборот – установить флаг Согласование через 1С:Документооборот.
  • Определить корневой каталог в 1С:Документообороте в котором будут храниться используемые файлы.
  • Настроить автоматическое заполнение реквизитов документа 1С:Документооборота из данных объекта, для которого создается документ (заполнить справочник Настройки заполнения объектов 1С:Документооборота).

Работа подсистемы

В режиме интеграции учетной системы с 1С:Документооборотом в последнем доступно решение следующих задач:

  • структурированное безопасное хранение произвольных типов документов и файлов, принадлежащих объектам учетной системы (офисные документы, изображения, архивы, аудио-видео данные, файлы систем проектирования и т.д),
  • оперативный доступ из учетной системы к документам и файлам размещенным в 1С:Документообороте с учетом прав пользователей,
  • возможность установки и редактирования связей между объектами учетной системы и зарегистрированными на их основании документами в 1С:Документообороте,
  • централизованное управление бизнес-процессами предприятия с возможностью использования иерархий - в отличие от возможностей типовой конфигурации, где формирование бизнес-процессов было доступно только для подсистем Продажи, а также Запасы и закупки, интеграция с 1С:Документооборотом позволяет расширить данную возможность для объектов всех подсистем,
  • индивидуальная маршрутизация документов при помощи настраиваемых шаблонов (установленных на предприятии правил работы с документами), например, для согласования договоров, сумма которых превышает 100 тысяч рублей, может быть использован один шаблон, а для согласования всех остальных договоров - другой.
  • возможность автоматического формирования истории электронной переписки по объектам конфигурации - при помощи использования встроенного в 1С:Документооборот почтового клиента.

При настроенной интеграции с 1С:Документооборотом для пользователей учетной системы доступен типовой перечень бизнес-процессов:

  • Рассмотрение – документ попадает на рассмотрение к руководителю и с его резолюцией возвращается к автору документа.
  • Исполнение – документ передается на исполнение всем пользователям по списку для соблюдения исполнительской дисциплины. Один из пользователей может быть назначен ответственным исполнителем.
  • Согласование – документы попадают на согласование указанным респондентам и потом возвращаются к автору бизнес-процесса для ознакомления с результатами согласования или отправки на повторное согласование.
  • Утверждение – документ попадает на утверждение к ответственному лицу и возвращается к автору для ознакомления с результатом утверждения.
  • Регистрация – документ попадает к секретарю для присвоения ему регистрационного номера, заверения печатью организации и отправки корреспонденту.
  • Ознакомление – бизнес-процесс предназначен для рассылки пользователям из списка, при этом автор не оповещается о результатах ознакомлений.
  • Поручение – бизнес-процесс предназначен для раздачи поручений сотрудникам и проверки своевременности исполнения.
  • Приглашение – используется для приглашения на различные мероприятия.
  • Обработка внутреннего документа – бизнес-процесс предназначен для обработки по заранее определенным правилам.
  • Обработка входящего документа – бизнес-процесс автоматизирует полный цикл обработки входящего документа – регистрация, рассмотрение, исполнение.
  • Обработка исходящего документа – бизнес-процесс автоматизирует полный цикл обработки исходящего документа – согласование, утверждение, регистрация.
  • Комплексный процесс – позволяет настроить произвольный маршрут обработки документа, состоящий из отдельных этапов.

Каждый бизнес-процесс по мере реализации формирует задачи, адресованные определенным пользователям. Формирование различного рода задач продиктовано необходимостью упорядочивания работ, а также контроля качества и своевременного исполнения бизнес-процессов. Причем, при формировании задачи не обязательно адресуются конкретным исполнителям. Например, документ может быть отправлен на утверждение директору, а система автоматически маршрутизирует задачу тому, кто в данный момент выполняет эту функцию – директору или его заместителю.
В рамках интеграции с 1С:Документооборотом управление задачами осуществляется пользователем непосредственно с рабочего стола информационной системы.

Из формы задачи доступно выполнение следующих действий:

  • перенаправление задач другому исполнителю;
  • принятие задачи к исполнению;
  • создание на основании задач новых бизнес-процессов;
  • переход в список всех задач бизнес-процесса текущей задачи.

При интеграции 1С:Документооборота с учетной системой ролевая доступность к объектам конфигурации регулируется настройками 1С:Документооборота.
